# 记账小程序前端开发总结

## 1. 项目结构

我为记账小程序创建了一个完整的前端项目，使用 Vue 3 + Vite 作为技术栈，包含以下核心文件和目录：

```
spring-finance-frontend/
├── src/
│   ├── router/          # 路由配置
│   ├── stores/          # 状态管理
│   ├── services/        # API 服务
│   ├── views/           # 页面组件
│   ├── components/      # 通用组件
│   ├── utils/           # 工具函数
│   ├── main.js          # 应用入口
│   └── App.vue          # 根组件
├── public/              # 静态资源
├── index.html           # HTML 模板
├── package.json         # 项目配置
└── vite.config.js       # Vite 配置
```

## 2. 实现的功能

### 2.1 核心页面

1. **首页** (`HomeView.vue`)
   - 财务概览：展示总收入、总支出和结余
   - 快捷操作：记一笔、统计、预算
   - 最近交易记录

2. **记账页面** (`RecordView.vue`)
   - 金额输入
   - 分类选择
   - 日期选择
   - 备注填写
   - 支持支出和收入记录

3. **统计页面** (`StatsView.vue`)
   - 按日/周/月/年的收支数据可视化
   - 收支趋势图表
   - 支出分类饼图
   - 收入分类饼图

4. **预算管理页面** (`BudgetView.vue`)
   - 设置各分类预算
   - 查看预算使用情况
   - 预算进度条展示
   - 支持编辑和删除预算

5. **账户管理页面** (`AccountsView.vue`)
   - 添加/编辑/删除账户
   - 支持多种账户类型（现金、银行卡、支付宝、微信等）
   - 账户余额展示
   - 账户颜色自定义

6. **个人中心页面** (`ProfileView.vue`)
   - 用户信息管理
   - 数据备份与恢复
   - 数据导出
   - 应用设置
   - 关于应用

### 2.2 技术实现

1. **状态管理**：使用 Pinia 管理应用状态，包括分类、支出、收入、预算和账户数据。
2. **路由管理**：使用 Vue Router 实现页面导航。
3. **API 服务**：封装与后端的通信逻辑，支持数据的增删改查。
4. **本地存储**：使用 localforage 实现数据的本地持久化，确保离线访问。
5. **数据可视化**：使用 ECharts 实现收支趋势和分类饼图。
6. **UI 设计**：使用 Vant 组件库和自定义样式，确保界面美观、交互流畅。
7. **响应式布局**：适配不同屏幕尺寸，确保在手机等小屏设备上的良好体验。

### 2.3 数据同步机制

- **在线模式**：优先从后端 API 获取数据，并同步到本地存储。
- **离线模式**：当网络不可用时，从本地存储读取数据，并在网络恢复后同步到后端。
- **数据备份**：支持将数据导出为 JSON 文件，以便备份和恢复。
- **数据导出**：支持将交易数据导出为 CSV 文件，便于在 Excel 等工具中分析。

## 3. 使用说明

### 3.1 开发环境运行

1. 进入前端项目目录：
   ```bash
   cd spring-finance-frontend
   ```

2. 安装依赖：
   ```bash
   npm install
   ```

3. 启动开发服务器：
   ```bash
   npm run dev
   ```

4. 在浏览器中访问：`http://localhost:5173/`

### 3.2 生产环境构建

1. 构建项目：
   ```bash
   npm run build
   ```

2. 将 `dist` 目录部署到服务器。

### 3.3 与后端集成

1. 确保后端服务（`spring-finance` 模块）已经启动，运行在 `http://localhost:8080`。
2. 前端会自动与后端 API 进行通信，同步数据。

### 3.4 核心功能使用

1. **记账**：点击底部导航栏的 "记账" 按钮，输入金额、选择分类、设置日期和备注，然后保存。
2. **查看统计**：点击底部导航栏的 "统计" 按钮，选择不同的时间范围，查看收支趋势和分类统计。
3. **管理预算**：在 "我的" 页面中进入 "预算管理"，添加或编辑各分类的预算。
4. **管理账户**：在 "我的" 页面中进入 "账户管理"，添加或编辑不同类型的账户。
5. **数据备份**：在 "我的" 页面中点击 "备份数据"，下载 JSON 格式的备份文件。
6. **数据恢复**：在 "我的" 页面中点击 "恢复数据"，选择之前下载的备份文件。

## 4. 技术特点

1. **模块化设计**：采用组件化和模块化的设计思路，代码结构清晰，易于维护和扩展。
2. **响应式布局**：适配不同屏幕尺寸，提供良好的移动端体验。
3. **数据持久化**：使用本地存储确保数据安全可靠，支持离线访问。
4. **可视化分析**：通过图表直观展示财务数据，帮助用户更好地理解自己的消费习惯。
5. **用户友好**：界面美观、交互流畅，操作简单直观。
6. **可扩展性**：代码结构设计合理，便于后续添加新功能。

2. **功能扩展**：
   - 添加用户认证系统，支持多用户。

## 6. 总结

本次开发完成了一个功能完整、界面美观、交互流畅的记账小程序前端，包含了收入/支出记录、账单统计、预算管理、账户管理和个人中心等核心功能。通过模块化设计和响应式布局，确保了应用在不同设备上的良好体验。同时，实现了本地数据存储与同步机制，确保用户数据安全可靠。
